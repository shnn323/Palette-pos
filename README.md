# LOVE FNC 후원 리워드 주문 포스 (실시간 재고 동기화)

손님이 직접 수량을 누르면 재고가 실시간으로 줄어들고, 계산완료를 누르면 다음 손님을 위해 화면이 초기화되는 주문 웹앱.
Supabase Realtime을 사용해서 누가 어디서 수량을 바꾸든 모든 접속자 화면에 거의 즉시 반영됨.

## 1단계 — Supabase 설정 (5분)

1. https://supabase.com 에서 무료 계정 생성 (깃허브 계정으로 가입 가능)
2. "New project" 클릭, 이름과 DB 비밀번호 설정 후 생성
3. 프로젝트 생성되면 왼쪽 메뉴 "SQL Editor" 클릭
4. 이 폴더 안의 `supabase_schema.sql` 파일 내용을 통째로 복사해서 붙여넣고 "Run" 클릭
   - 상품 테이블 생성, 재고 차감/복구 함수, 실시간 동기화 설정, 초기 상품 데이터(개인 포토카드·그룹세트·굿즈·뽑기판)까지 한번에 들어감
5. 왼쪽 메뉴 "Project Settings" → "API" 들어가서 다음 두 개 복사해두기:
   - `Project URL`
   - `anon public` 키

## 2단계 — 로컬에서 키 설정

1. 이 폴더의 `.env.example` 파일을 복사해서 `.env` 라는 이름으로 저장
2. 방금 복사한 값들을 넣기:

```
VITE_SUPABASE_URL=복사한 Project URL
VITE_SUPABASE_ANON_KEY=복사한 anon public 키
```

(관리탭이 없는 버전이라 VITE_ADMIN_PASSWORD는 더 이상 필요 없음.)

## 3단계 — 로컬 실행해서 확인 (선택)

터미널에서 이 폴더로 이동 후:

```
npm install
npm run dev
```

터미널에 뜨는 주소(보통 http://localhost:5173)로 접속해서 수량 누르면 재고가 바뀌는지, 계산완료 누르면 초기화되는지 확인.

## 4단계 — Vercel로 배포

1. 이 폴더를 깃허브 저장소에 올리기
2. https://vercel.com 가입 (깃허브 계정으로)
3. "Add New Project" → 방금 올린 저장소 선택
4. "Environment Variables" 항목에 `.env`에 넣었던 2개 값을 등록
   (VITE_SUPABASE_URL, VITE_SUPABASE_ANON_KEY)
5. "Deploy" 클릭, 1~2분 후 공개 URL 생성됨

이제 그 URL에 누구나 접속해서 상품을 보고 수량을 누를 수 있고, 한 사람이 수량을 바꾸면 다른 모든 접속자 화면의 재고도 거의 즉시 바뀜.

## 구성된 상품

- 개인 포토카드: 1장 10,000원, 재고 제한 없음
- 포토카드 그룹 세트 + 피크키링: FTISLAND 200개, N.Flying 250개, Inseong 40개, Hi-Fi Un!corn 30개, AxMxP 40개
- 굿즈: 반다나 200개, 슬로건 100개, 스티커 200개
- 뽑기판: 1회 5,000원, 1000개

수량이나 가격, 재고를 나중에 바꾸고 싶으면 Supabase 대시보드의 "Table Editor" → products 테이블에서 직접 셀을 클릭해 수정하면 됨 (별도 관리 화면 없이도 거기서 바로 가능).

## 참고

- "계산완료" 버튼은 화면에 보이는 수량 카운터만 0으로 초기화함. 이미 차감된 재고는 그대로 유지되니 다음 손님 주문을 새로 받을 수 있음.
- 동시에 여러 사람이 같은 상품을 거의 동시에 선택해도 재고가 마이너스로 안 떨어지게 DB 함수(`sell_product`)로 막아뒀음.
- 화면은 누르자마자 바로 반영(낙관적 업데이트)되고, 서버 저장은 뒤에서 처리돼서 체감 속도가 빠름. 만약 마지막 1개를 두 사람이 동시에 누르면, 나중에 실패한 쪽 화면에 품절 안내가 뜨고 자동으로 되돌아감.
- Vercel, Supabase 둘 다 무료 플랜으로 시작 가능.
